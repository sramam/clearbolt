# V1.5 — Broker-direct rollout + telemetry hardening

V1.5 extends the production foundation with the **full broker-direct ingestion lane** ([ADR 0016](../decisions/0016-broker-direct-ingestion-lane.md)) and several operational items deferred from V1.

## Goal

Enumerate brokers from franchise networks and state license databases, crawl allow-listed broker websites at scale, and measure whether broker-direct adds net-new deals beyond marketplace coverage (falsifiability criteria in ADR 0016).

## Scope

### Broker-direct (Part B completion)

- **Directory adapters:** `sunbelt`, `transworld`, `state-dre/{ca,fl,az}` in [`packages/broker-directory`](../../packages/broker-directory/agents.md).
- **Broker-site plugins:** franchise bespoke parsers for Sunbelt and Transworld listing index pages ([`packages/broker-site`](../../packages/broker-site/agents.md) `BrokerSiteParserPlugin`).
- **Allow-list expansion:** operator workflow to add domains after robots + ToS review; per-source ToS table kept current in broker-directory `agents.md`.
- **Pocket-listing freshness:** policy finalized ([open.md](../decisions/open.md)) and enforced in dedup confidence + `DealEvent.freshnessWarning` thresholds.

### Marketplace broker enrichment (Part A hardening)

- BizQuest + BusinessesForSale + LoopNet broker materialization tests green alongside BizBuySell.
- BizBuySell + BizQuest broker **directory** catalog walks production-ready (legal sign-off required for BBS).

### Telemetry / ops (from V1 deferrals)

- OTel log + trace backend choice ([open.md](../decisions/open.md)).
- VictoriaMetrics topology decision.
- PostHog tenancy model for cross-workspace queries.
- Cross-workspace de-identification pipeline (Layer 3 aggregates).

## Acceptance criteria

- Sunbelt and Transworld directory enumeration yields ≥100 `Broker` rows with `websiteDomain` populated in a pilot region.
- Broker-site crawl on allow-listed franchise domains ingests listings that dedup-merge with marketplace records when the same deal exists on BizBuySell.
- ADR 0016 falsifiability measurement runs for 30 days; decision to continue or defer broker-direct documented.
- At least one state-DRE adapter (`state-dre/ca`) ships with documented source URL and ToS row.

## Validation criteria

- **Given** Sunbelt location fixtures, **when** `sunbelt` `discoverBrokers` runs, **then** candidates include `firmName` and office `state`. Coverage: unit. Test: `packages/broker-directory/tests/sunbelt.test.ts`.
- **Given** the Sunbelt plugin registered, **when** `BrokerSiteCrawler` runs on a Sunbelt host fixture, **then** listings are parsed without invoking `broker-site-extract`. Coverage: unit. Test: `packages/broker-site/tests/sunbelt-plugin.test.ts`.
- **Given** CA DRE fixture data, **when** `state-dre/ca` discovery runs, **then** each candidate has license-derived `externalId`. Coverage: unit. Test: `packages/broker-directory/tests/state-dre-ca.test.ts`.
- **Given** a pocket listing ingested only via `broker-site`, **when** no marketplace `SourceRecord` exists after 14 days, **then** `freshnessWarning` policy matches the documented rule in `packages/dedup/agents.md`. Coverage: integration. Test: `packages/dedup/tests/pocket-listing-freshness.test.ts`.
- **Given** 30 days of production ingest metrics, **when** ADR 0016 measurement script runs, **then** it outputs continue/defer recommendation with net-new deal percentage. Coverage: smoke. Test: `scripts/broker-direct-coverage-report.mjs`.

## Out of scope (V2)

- Outreach using broker contact data ([V2.md](V2.md)).
- Full off-market source expansion beyond broker-direct (Google Maps, SOS, etc.) — remains per-source ADRs in [open.md](../decisions/open.md).
