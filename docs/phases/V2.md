# V2 — Full searcher workspace

V2 builds out the multi-user, outreach, and full-diligence experience on the V1 production foundation.

## Scope

### Multi-user collaboration

- Roles/invites (`owner`, `admin`, `member`, `viewer`).
- Shared workspace state: notes, mentions, activity feed.
- Audit log surfaced in workspace UI.
- SSO (SAML / OIDC) for team accounts.

### Browser extension build

- Manifest v3 extension built and shipped per [../../apps/extension/PLAN.md](../../apps/extension/PLAN.md).
- Per-host heuristics for Axial, BizBuySell detail, ChatGPT, Claude.ai, Gemini, Perplexity, generic fallback.
- Side panel review/edit UX.
- Posts to the V1 `POST /api/captures` Cloudflare Worker.

### Off-market lead management with outreach

- Email/phone outreach.
- Suppression lists, opt-outs, consent tracking.
- Email infra warmed (separate sending domains for transactional vs outreach).
- Touch history, replies, bounces, next actions.

### Listing history and change tracking surfaced in UI

- Per-deal `DealEvent` log visible in the deal page.
- Diffs for material changes (price, status, broker).
- Notifications wired to material-change events.

### Comps and valuation views

- Asking-multiple comps by industry / geography / size band.
- User-curated comps.
- Listing-vs-comps view per deal.

### Document and diligence layer

- Per-deal document store with type, source, access scope.
- CIM text extraction, PDF P&L tables.
- NDA tracking (status, signed date, expiry).
- Diligence checklist generated from `DealQualityScore` gaps.

### Ranking informed by personalization, market fit, financing scenarios

- Workspace ranking profile learns from likes/dislikes/passes.
- Financing scenarios feed affordability ranking.
- Explainable ranking reasons.

### Eval harness for AI features

- Golden sets per task (extraction, dedup-pair, market-definition draft, quality-of-deal score, outreach draft, wiki ingest, wiki query).
- Regression suite as a CI gate for high-impact tasks.
- Live sampling for drift detection.

### Container sandboxes for the agent harness

- `local-node`, `daytona`, `e2b`, `vercel-sandbox` adapters available for skills/tasks that need real shell execution.
- Default sandbox remains `virtual` (in-process) for cheap, predictable tool use.

### Cloudflare Queues adapter for the queue contract (optional)

- If we want CF-side queue producers without round-tripping to Fly's pg-boss, ship a CF Queues backend.

### Full federated observability UI (Layer 4)

Replaces V1's minimal `/admin` (embedded PostHog + VM proxy) with native React panels per [telemetry.md §Layer 4](../architecture/telemetry.md#layer-4--observability-ui):

- Native panels for VM (metrics), the V1.5-chosen log/trace backend, and PostHog — all rendered in the app's design system.
- Cross-source linking: click a span → see related product events → see related dedup decisions, in one workflow.
- Saved views per workspace; shareable URLs.
- Per-workspace cost dashboards with explicit "your spend, your AI cost, your storage cost" breakdown — searcher-facing transparency.
- Operators never leave the product to investigate.

### Cross-workspace de-identification pipeline (V1.5 work; landed in V2)

- Nightly `services/deidentify` job strips workspaceId/userId/free-text/sensitive numerics, buckets numerics, geographies (MSA), industries (NAICS-2).
- Anonymous PostHog project drives product roadmap decisions and prompt-eval cohort analysis.
- Workspace owners opt out via workspace settings; default opt-in for new workspaces with clear copy.
- DSR (data subject rights) job: delete-by-userId across PostHog and log backend within 30 days.

## Out of scope (V3+)

- Provider profiles and deal-team shopping (brokers, lenders, lawyers, CPAs, QoE) — see [V3-plus.md](V3-plus.md).
- Lender prequalification integrations.
- Public API, webhooks, CRM integrations (HubSpot, Salesforce, Affinity).
- Cross-workspace shared comps/research with privacy controls.
- Multi-region/currency support beyond US.

## Validation criteria

V2 acceptance is met when each item in [Scope](#scope) above maps to a passing test:

### Functional
- **Given** a workspace with multiple invited users, **when** a `member` opens a deal, **then** they see the same canonical deal as the `owner` (subject to role-gated fields). Coverage: integration. Test: `apps/web/tests/multi-user-deal-view.test.ts`.
- **Given** the V2 browser extension installed, **when** the user clicks "Save to CRM" on an Axial teaser page, **then** within 5 seconds a `WorkspaceCapture` exists in their workspace with raw HTML stored in `EvidenceStore`, markdown stored, and a wiki page proposed. Coverage: smoke. Test: `apps/extension/tests/axial-capture-end-to-end.smoke.test.ts`.
- **Given** a contact on a workspace's outreach list, **when** an email is sent via the outreach UI, **then** suppression list, opt-out, and consent checks run before send and the touch is recorded in the `OutreachThread`. Coverage: integration. Test: `services/outreach/tests/send-with-guards.test.ts`.
- **Given** a deal whose price changes between two scrapes, **when** the change is detected, **then** a `DealEvent` is created, the deal page UI surfaces a diff, and (if the workspace subscribes) a notification is sent. Coverage: integration. Test: `apps/web/tests/deal-event-diff.test.ts`.
- **Given** a workspace with the V2 ranking enabled, **when** a user likes 5 listings in industry X, **then** the next saved-search run promotes industry-X listings (verifiable A/B vs the unranked baseline). Coverage: golden-set. Test: `services/ranking/tests/personalization-fixture.test.ts`.

### Eval harness
- **Given** the V2 eval harness, **when** the regression suite runs against the wiki-ingest, wiki-query, market-definition, and quality-of-deal tasks, **then** golden-set pass rate ≥ the previously-recorded baseline minus a configurable drift tolerance. Coverage: golden-set + CI gate. Test: `services/evals/tests/regression-suite.test.ts`.

### Observability UI (full federated)
- **Given** a workspace `admin` opens `/admin` in V2, **when** they click on a span in the trace panel, **then** the related Layer 3 product events for that `traceId` and the related Layer 1 metrics are linked one click away — all in the native UI, no vendor console hop. Coverage: integration. Test: `apps/web/tests/admin-federated-cross-link.test.ts`.

### Cross-workspace privacy
- **Given** the V1.5 de-identification pipeline ships in V2, **when** a workspace owner opts out, **then** their workspace events disappear from the anonymous PostHog project within one nightly run. Coverage: integration. Test: `services/deidentify/tests/opt-out-honored.test.ts`.

### Sandboxes
- **Given** a skill that requires shell execution (V2 container sandbox), **when** the harness runs it with `sandbox: 'daytona'` or `sandbox: 'e2b'`, **then** the skill's stdout/stderr/files are captured and returned per the harness contract. Coverage: integration. Test: `packages/agents-sandboxes/tests/daytona-roundtrip.test.ts` and `::e2b-roundtrip.test.ts`.
